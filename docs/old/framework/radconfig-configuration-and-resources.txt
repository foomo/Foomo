RadConfig ensures, that you access resources following the conventions of the \
framework as defined in the hierarchy.


+++ Retrieving configurations

See the configuration chapter.

<geshi type="php">

RadConfig::getConf($module, $domain, $subDomain)

</geshi>

+++ Checking the run mode

<geshi type="php">

// find out which run mode you are in
$runMode = RadConfig::getMode();

// and behave accordingly
if($runMode != RadConfig::MODE_PRODUCTION) {
	trigger_error('we can be quite verbose', E_USER_NOTICE);
}

</geshi>

+++ Finding the right place for resources

++++ Data

<geshi type="php">

// a place to put data to
$varDir = RadConfig::getVarDir();

</geshi>


++++ Logs

<geshi type="php">

// a place to log
RadConfig::getLogDir($module);

</geshi>

++++ Temporary data

<geshi type="php">

// the place to put temporary data
RadConfig::getTempDir();

</geshi>
